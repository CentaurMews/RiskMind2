function getThemeColors() {
  const style = getComputedStyle(document.documentElement);
  const getCssVar = (name: string) => style.getPropertyValue(name).trim();
  return {
    severityCritical: hslStringToColor(getCssVar("--severity-critical")),
    severityHigh: hslStringToColor(getCssVar("--severity-high")),
    severityMedium: hslStringToColor(getCssVar("--severity-medium")),
    severityLow: hslStringToColor(getCssVar("--severity-low")),
    // ...more vars
  };
}

    **Data transformation (client-side per D-09):**
    - Flatten incoming risks array into parallel data rows
    - Each row = [likelihood, impact, score, categoryIndex, statusIndex, ownerRiskCount]
    - Score = likelihood * impact (D-09)
    - Category: map to numeric index for axis (technology=0, operational=1, compliance=2, financial=3, strategic=4, reputational=5)
    - Status: map to numeric index (open=0, mitigated=1)
    - Risk Count per Owner: group risks by category, count per category, assign count to each risk in that category (derived dimension per D-01)

    **6 Parallel Axes (D-01) — order for maximum visual insight:**
    1. Likelihood (1-5, numeric)
    2. Impact (1-5, numeric)
    3. Score (1-25, numeric, computed)
    4. Category (categorical: Cyber, Ops, Compliance, Financial, Strategic, Reputational)
    5. Status (categorical: Open, Mitigated)
    6. Risk Count per Category (numeric, derived)

    **Line coloring (D-02):**
    - Use `getThemeColors()` pattern from risk-heatmap-chart.tsx (copy `hslStringToColor` and `getThemeColors` helpers)
    - Color each line by severity derived from score: critical (>=15), high (>=10), medium (>=5), low (<5)
    - Use `--severity-critical`, `--severity-high`, `--severity-medium`, `--severity-low` CSS variables

    **Line styling (D-10, D-11):**
    - Default line opacity: 0.35 (muted, beautiful flowing pattern)
    - When brush is active: selected lines opacity 0.9, unselected lines opacity 0.05 (near-invisible)
    - No heavy gridlines. Axis labels at top. Minimal chrome.
    - Line width: 1.5px default, 2.5px on emphasis

    **Interactive axis brushing (D-03):**
    - Use ECharts `parallelAxis` with `areaSelectStyle` for brush interaction
    - Configure `parallel.parallelAxisDefault.areaSelectStyle` with subtle styling
    - Set `brushType: 'interval'` on numeric axes, `brushType: 'interval'` on categorical axes
    - When brushing one axis, all lines not matching fade via ECharts built-in parallel filtering

    **Tooltip (D-04):**
    - On line hover, show tooltip with: Risk title, Score (L x I), Category, Status
    - Use ECharts `tooltip` with custom formatter

    **Click navigation (D-05):**
    - On line click, call `onRiskClick(riskId)` prop
    - Store risk ID mapping: maintain a `riskIdMap` array parallel to the data array so data index maps back to risk.id

    **Dark mode (same pattern as risk-heatmap-chart.tsx):**
