Create `src/adapters/sentinel.ts` implementing `SignalFeedAdapter`:

**CRITICAL:** Do NOT use `@azure/arm-securityinsight`. The SIEM agent was retired November 2025. Use the Azure Log Analytics REST API directly with `@azure/identity` for OAuth2 tokens.

1. Import `ClientSecretCredential` from `@azure/identity`.

2. Helper `getLogAnalyticsToken(config: SentinelConfig)`:
   - Create `new ClientSecretCredential(config.azureTenantId, config.clientId, config.clientSecret)`
   - Call `credential.getToken("https://api.loganalytics.io/.default")`
   - Return the token string

3. Helper `queryLogAnalytics(config: SentinelConfig, kql: string)`:
   - Get token via helper above
   - POST to `https://api.loganalytics.io/v1/workspaces/${config.workspaceId}/query`
   - Headers: `Authorization: Bearer ${token}`, `Content-Type: application/json`
   - Body: `{ query: kql }`
   - Parse response: `data.tables[0]` has `columns` (array of {name, type}) and `rows` (array of arrays). Map rows to objects using column names as keys.
   - Return array of incident objects

4. `poll(config: SentinelConfig, since: Date)`:
   - KQL query: `SecurityIncident | where TimeGenerated > datetime(${since.toISOString()}) | project IncidentNumber, Title, Description, Severity, Status, Classification, CreatedTime, ClosedTime, Owner, AlertIds | order by TimeGenerated desc | take 500`
   - For each incident row, create RawSignal:
     - `content`: `"[Sentinel] ${severity} incident: ${title}. ${description}"`
     - Normalize for hash: `JSON.stringify({ incidentNumber, title })` — incident number is stable
     - `contentHash`: `computeContentHash(normalized)`
     - `externalId`: `sentinel-${incidentNumber}` — dedup by incident number
     - `metadata`: `{ incidentNumber, title, description, severity, status, classification, createdTime, closedTime, owner, alertIds }`
     - `sourceEventTimestamp`: parse `createdTime` as Date
   - Return RawSignal array

5. `testConnection(config: SentinelConfig)`:
   - Try `queryLogAnalytics(config, "SecurityIncident | take 1")`
   - If succeeds: `{ ok: true, message: "Connected to Log Analytics workspace. Sentinel incidents accessible." }`
   - If fails: `{ ok: false, message: "Failed to connect: ${error.message}. Verify Tenant ID, Client ID, Client Secret, and Workspace ID. Ensure app registration has Log Analytics Reader role." }`

6. Register: `adapters["sentinel"] = sentinelAdapter;`

Create `src/adapters/misp.ts` implementing `SignalFeedAdapter`:

1. Helper `mispFetch(config: MispConfig, path: string, body?: object)`:
   - URL: `${config.baseUrl.replace(/\/+$/, '')}${path}`
   - Headers: `Authorization: ${config.apiKey}`, `Accept: application/json`, `Content-Type: application/json`
   - Method: body ? "POST" : "GET"
   - Return parsed JSON

2. `poll(config: MispConfig, since: Date)`:
   - Build search body for `/events/restSearch`:
     ```json
     {
       "timestamp": Math.floor(since.getTime() / 1000),
       "limit": 500,
       "page": 1,
       "includeAttributes": true
     }
     ```
   - If `config.eventFilters?.minThreatLevel`, add `threat_level_id` filter (1=High, 2=Medium, 3=Low, 4=Undefined — filter <=)
   - If `config.eventFilters?.tags`, add `tags` filter array
   - POST to `/events/restSearch` with the search body
   - For each MISP event in `response.response`:
     - Extract event `id`, `info` (title), `threat_level_id`, `date`, `Org.name`, `Attribute` array
     - Group attributes by type: IP addresses, domains, hashes, CVE IDs, email addresses, URLs
     - Create RawSignal:
       - `content`: `"[MISP] ${event.info} (Threat Level: ${threatLevelLabel}). IoCs: ${iocSummary}"`
       - Normalize for hash: `JSON.stringify({ eventId: event.id, info: event.info })` — event ID is stable
       - `contentHash`: `computeContentHash(normalized)`
       - `externalId`: `misp-${event.id}`
       - `metadata`: `{ eventId: event.id, info: event.info, threatLevel: threatLevelId, orgName, date, attributes: groupedAttributes, attributeCount: event.Attribute?.length ?? 0, tags: event.Tag?.map(t => t.name) }`
       - `sourceEventTimestamp`: parse `event.date` as Date
   - Return RawSignal array

3. `testConnection(config: MispConfig)`:
   - GET `/users/view/me` (MISP auth check endpoint)
   - If 200: `{ ok: true, message: "Connected to MISP instance at ${config.baseUrl}. Org: ${data.User?.org_id ?? 'unknown'}." }`
   - If fails: `{ ok: false, message: "Failed to connect to MISP: ${error.message}" }`

4. Register: `adapters["misp"] = mispAdapter;`

**Threat level mapping for display:** 1="High", 2="Medium", 3="Low", 4="Undefined".

**Attribute type grouping:** Group by MISP attribute `type` field:
- IPs: type in ["ip-src", "ip-dst"]
- Domains: type in ["domain", "hostname"]
- Hashes: type in ["md5", "sha1", "sha256"]
- CVEs: type === "vulnerability"
- Emails: type in ["email-src", "email-dst"]
- URLs: type === "url"
