Create `src/adapters/nvd.ts` implementing `SignalFeedAdapter`:

1. Import `pRateLimit` from `p-ratelimit`. Create limiter: `pRateLimit({ interval: 30_000, rate: 45, concurrency: 1 })` — stays under NVD's 50 req/30s limit with headroom.

2. `poll(config: NvdConfig, since: Date)`:
   - Build URLSearchParams with `lastModStartDate` = `since.toISOString()` and `lastModEndDate` = `new Date().toISOString()`.
   - If `config.keywords` has entries, add `keywordSearch` param (comma-joined).
   - Paginate: fetch page 0 with `startIndex=0`, `resultsPerPage=2000`. Read `totalResults` from response. Loop with startIndex += 2000 until all pages fetched.
   - For each CVE vulnerability in response:
     - Extract `cveId` from `cve.id`
     - Extract CVSS v3.1 base score from `cve.metrics?.cvssMetricV31?.[0]?.cvssData?.baseScore` (fallback to v3.0, then v2)
     - Extract English description from `cve.descriptions.find(d => d.lang === 'en')?.value`
     - Normalize content: `JSON.stringify({ id: cveId, cvssV3: baseScore, description })` — this ensures identical CVEs hash identically across polls
     - Compute `contentHash` via `computeContentHash(normalizedContent)`
     - Set `externalId` to `cveId` (e.g., "CVE-2024-12345")
     - Set `metadata` to `{ cveId, cvssV3BaseScore: baseScore, cvssVector, publishedDate, lastModifiedDate, references, affectedProducts }`
     - Set `content` to human-readable: `"[CVE-ID] (CVSS X.X) Description text"`
   - Return array of `RawSignal[]`
   - Use the rate limiter wrapper on every fetch call

3. `testConnection(config: NvdConfig)`:
   - Fetch `https://services.nvd.nist.gov/rest/json/cves/2.0?resultsPerPage=1` with API key header if provided.
   - If 200, return `{ ok: true, message: "NVD API reachable. API key: ${config.apiKey ? 'configured' : 'not set (rate-limited)'}." }`
   - On error, return `{ ok: false, message: error.message }`

4. Create and export `nvdAdapter` instance. Register in `types.ts`: add `import { nvdAdapter } from "./nvd"; adapters["nvd"] = nvdAdapter;` at bottom of types.ts OR register via a separate `src/adapters/index.ts` barrel that imports all adapters and populates the registry. Choose whichever is cleaner — but the `adapters` record in types.ts must be populated.

**API details:**
- NVD endpoint: `https://services.nvd.nist.gov/rest/json/cves/2.0`
- Headers: `{ apiKey: config.apiKey }` if apiKey provided (omit header if not)
- Response shape: `{ resultsPerPage, startIndex, totalResults, vulnerabilities: [{ cve: { id, descriptions, metrics, references, configurations } }] }`

Create `src/adapters/shodan.ts` implementing `SignalFeedAdapter`:

1. Import `pRateLimit` from `p-ratelimit`. Create limiter: `pRateLimit({ interval: 1_000, rate: 1, concurrency: 1 })` — Shodan free tier: 1 req/sec.

2. `poll(config: ShodanConfig, since: Date)`:
   - Query all vendors from the database that have a `website` field (or domain-like field). Extract domain from website URL.
   - For each vendor domain:
     a. **DNS resolve**: call `https://api.shodan.io/dns/resolve?hostnames=${domain}&key=${config.apiKey}` to get IP addresses. Skip vendor if no IPs resolve.
     b. **Host query**: for each resolved IP, call `https://api.shodan.io/shodan/host/${ip}?key=${config.apiKey}`. Handle 404 (no data) gracefully — skip.
     c. From host response, extract: `ports`, `data` array (each entry has `port`, `transport`, `product`, `version`, `vulns`).
     d. For each host result, create a RawSignal:
        - `content`: human-readable summary: `"Shodan scan for {domain} ({ip}): {N} open ports [{port list}]. Services: {service list}. CVEs: {vuln list or 'none'}"`
        - Normalize for hash: `JSON.stringify({ ip, ports: sorted_ports, vulns: sorted_vulns })` — sort arrays for deterministic hashing
        - `contentHash`: computeContentHash of normalized content
        - `externalId`: `shodan-${ip}` (one signal per IP per poll — updated on next poll if fingerprint changes)
        - `vendorId`: the vendor's UUID
        - `metadata`: `{ ip, domain, ports, services: [{port, transport, product, version}], vulns: [...cveIds], lastUpdate: host.last_update }`
   - Return all RawSignals collected across all vendors
   - Rate-limit every fetch call through the limiter

3. `testConnection(config: ShodanConfig)`:
   - Call `https://api.shodan.io/api-info?key=${config.apiKey}`
   - If 200, return `{ ok: true, message: "Shodan API key valid. Plan: ${data.plan}. Scan credits: ${data.scan_credits}." }`
   - On error (401 typically), return `{ ok: false, message: "Invalid Shodan API key" }`

4. Register: `adapters["shodan"] = shodanAdapter;` in the adapter registry.

**Pitfall note (from RESEARCH.md):** Shodan queries by IP, NOT domain. Must DNS-resolve first via `/dns/resolve`. Content hash must normalize (sort) ports and vulns arrays for deterministic dedup — otherwise array order changes cause false "new" signals.
