<!-- API endpoints from Plan 02 -->
GET /v1/assessment-templates — returns AssessmentTemplate[] (with isPrebuilt derived from description prefix)
GET /v1/assessment-templates/:id — returns single template
POST /v1/assessment-templates — body: { title, description?, questions: AssessmentTemplateQuestions, contextType }
PATCH /v1/assessment-templates/:id — body: partial update
DELETE /v1/assessment-templates/:id — 400 if pre-built
POST /v1/assessment-templates/:id/clone — returns new template

```typescript
export interface BranchCondition { questionId, operator: "equals"|"contains"|"greater_than", value, action: "show"|"hide" }
export interface QuestionOption { id, label, value, numericValue? }
export interface AssessmentQuestion { id, sectionId, text, type: "text"|"boolean"|"multiple_choice"|"numeric", weight, required, options?, numericMin?, numericMax?, conditions: BranchCondition[], isAiGenerated?, triggeredByQuestionId? }
export interface AssessmentSection { id, name, order, questions: AssessmentQuestion[] }
export interface AssessmentTemplateQuestions { sections: AssessmentSection[], version: number }
```

    **1. Create ConditionBuilder.tsx:**

    A row-based condition editor component. Props: `{ conditions: BranchCondition[], availableQuestions: AssessmentQuestion[], onChange: (conditions: BranchCondition[]) => void }`.

    Each condition row renders:
    - Select for "Question" — shows available questions (filtered to questions before the current one in the section, so you can only branch on prior questions). Display question text truncated to 40 chars.
    - Select for "Operator" — options: "equals", "contains", "greater than"
    - Input for "Value" — text input for the comparison value
    - Switch for "Action" — labeled "Show" / "Hide". Default "show".
    - X button to remove the condition row.

    "Add Condition" link/button below existing conditions to add a new empty condition.

    All changes immediately propagated to parent via onChange.

    **2. Create QuestionRow.tsx:**

    Props: `{ question: AssessmentQuestion, index: number, availableQuestions: AssessmentQuestion[], onUpdate: (q: AssessmentQuestion) => void, onRemove: () => void, onDragStart: (e) => void, onDragOver: (e) => void, onDrop: (e) => void }`.

    Render as a Card with:
    - Drag handle: GripVertical icon (lucide-react, h-4 w-4, text-muted-foreground) on the left side, 44px tall hit area. Set `draggable="true"` on the card, wire onDragStart/onDragOver/onDrop.
    - Collapsed state: question text (truncated, text-sm), type Badge (variant="outline", text is question.type), weight as "(w: {weight})", expand chevron.
    - Expanded state (toggle via ChevronDown/ChevronUp): full edit form:
      - Input for question text (label "Question")
      - Select for type: text, boolean, multiple_choice, numeric
      - Input for weight (type="number", min=0 max=10, step=1)
      - Checkbox for required
      - If type="multiple_choice": list of option inputs — each with label, value, numericValue (0-1). "Add Option" button.
      - If type="numeric": Input for numericMin, Input for numericMax.
      - ConditionBuilder component for branching conditions (pass availableQuestions filtered to only prior questions in the section).
    - Trash icon button to remove question (calls onRemove).

    Drag styling: `opacity-50` when dragging. Drop indicator: 2px solid primary horizontal rule between rows.

    **3. Create SectionBlock.tsx:**

    Props: `{ section: AssessmentSection, allSections: AssessmentSection[], onUpdate: (s: AssessmentSection) => void, onRemove: () => void }`.

    Render:
    - Section name Input (text-lg font-semibold placeholder="Section name").
    - Separator below name.
    - Ordered list of QuestionRow components for section.questions.
    - "Add Question" Button (variant="outline", Plus icon) at bottom of question list.
    - Adding a question creates a new AssessmentQuestion with: `{ id: crypto.randomUUID(), sectionId: section.id, text: "", type: "text", weight: 5, required: true, conditions: [] }`.

    Drag-to-reorder: manage question order within the section. On drop, reorder the questions array and call onUpdate.

    If section has 0 questions: show inline Alert — "No questions added yet."

    Small Trash2 icon button next to section name to remove entire section (with confirmation if section has questions).

    **4. Create builder.tsx:**

    Page component for `/assessments/templates/new` and `/assessments/templates/:id/edit`.

    Detect mode from URL: if `:id` param present, fetch template via GET /v1/assessment-templates/:id and populate form. Otherwise start empty.

    **Two-panel layout:**
    - Container: `flex gap-8` (on desktop), single column on mobile.
    - Left panel (60%): `w-3/5` on lg+.
      - Breadcrumb: Assessments > Templates > {title or "New Template"}
      - Input for template name (text-2xl font-semibold, placeholder "Template name")
      - Textarea for description (placeholder "Describe this template...")
      - Select for contextType: "vendor" or "framework"
      - "Add Section" Button below metadata fields.
      - List of SectionBlock components. Adding a section creates: `{ id: crypto.randomUUID(), name: "", order: sections.length, questions: [] }`.
      - "Save Template" Button (variant="default") fixed at bottom. On click:
        - Build AssessmentTemplateQuestions: `{ sections, version: 1 }` (or increment version on edit)
        - POST /v1/assessment-templates (new) or PATCH /v1/assessment-templates/:id (edit)
        - On success: toast "Template saved", navigate to /assessments/templates
        - On error: toast "Template couldn't be saved. Check required fields and try again." (destructive)
    - Right panel (40%): `w-2/5` on lg+, hidden below 768px.
      - Header: "PREVIEW" label (text-xs uppercase tracking-widest text-muted-foreground)
      - Read-only rendering of current template as it would appear during an assessment. Render each section with its questions. For each question, render the appropriate answer component (RadioGroup for boolean, RadioGroup for MC, Textarea for text, Slider for numeric) but disabled/non-interactive.
      - Updates live as user edits left panel.
      - On mobile: hidden by default. "Preview" button in header opens preview as Sheet (side panel overlay).

    **State management:**
    - Template state as `useState<{ title, description, contextType, sections: AssessmentSection[] }>`.
    - All edits update this state. Preview renders from same state.
    - No auto-save — explicit "Save Template" button.
