| Role | Size | Weight | Line Height |

| Body | 14px | 400 | 1.5 |
| Label | 14px | 500 | 1.4 |
| Heading | 20px | 600 | 1.2 |

Notes:
- Score numerals (e.g. "84 / 100") rendered at 28px weight 600 inside the gauge/donut chart center text.

- AI analysis narrative in results card uses 14px / 400, 1.5 line-height.

## Page Inventory

Three new pages required for this phase:

### 1. Template Library — `/assessments/templates`

- **Layout:** Standard AppLayout. Page header: "Assessment Templates" h1 (28px/600) + "New Template" primary button top-right.
- **Filter bar:** Two toggle buttons — "All" / "Vendor" / "Compliance" / "Incident". Renders inline below header.
- **Grid:** 3-column responsive grid (2-col tablet, 1-col mobile). Each card: template name (16px/600), type Badge, question count + section count as muted metadata (14px/400), description text (14px/400, 2-line clamp). Two actions per card: "Use Template" (primary Button), "Clone & Edit" (outline Button).
- **Pre-built templates** distinguished with a "Built-in" Badge in muted style. Built-in templates have no delete action.
- **Empty state:** Heading "No templates yet", body "Create your first assessment template to start evaluating vendors and compliance controls.", CTA "New Template".
- **Loading:** 6 Skeleton cards in same 3-column grid.

### 2. Template Builder — `/assessments/templates/new` and `/assessments/templates/:id/edit`

- **Layout:** Standard AppLayout with Breadcrumb. Two-panel layout: left panel (60% width) = builder canvas; right panel (40% width) = preview pane.
- **Left panel (Builder canvas):**
  - Template name Input + description Textarea at top (full width).
  - "Add Section" button below metadata fields.
  - List of SectionBlock components. Each SectionBlock: section name Input + ordered list of QuestionRow + "Add Question" button.
  - QuestionRow expanded state shows: question text Input, type Select, weight Input (0–10), branching conditions list via ConditionBuilder rows, "Add Condition" link.
  - "Save Template" primary button fixed at bottom of left panel.
- **Right panel (Preview pane):**
  - Labeled "Preview" in 14px/500 muted text at top.
  - Read-only rendering of current template as it would appear during an assessment session. Updates live as user edits left panel.
  - Preview uses same question rendering components as the assessment session to guarantee visual consistency.
- **Mobile:** Preview panel hidden below 768px. "Preview" toggle button appears in header to open as Sheet.

### 3. Assessment Session — `/assessments/:id/session`

- **Layout:** Standard AppLayout with Breadcrumb. Single-column centered layout, max-width 720px, horizontally centered.
- **Header:** Assessment subject name (vendor or framework name) + assessment template name below in muted text. WizardStepper below header.
- **Progress bar:** shadcn Progress component showing section completion %. "Section X of Y" label right-aligned.
- **Question area:** Ordered list of question items for the current section. Each question:
  - Question text at 14px/400.
  - Answer input appropriate to question type (RadioGroup for boolean, RadioGroup for multiple-choice, Textarea for text, Slider + numeric Input for numeric).
  - Weight shown as muted "(weight: X)" inline with question text.
  - AiFollowUpQuestion appears immediately after the triggering question when AI generates a follow-up. Fade-in 200ms.
- **Navigation:** "Previous" (ghost Button, left) and "Next Section" (primary Button, right) fixed at bottom of viewport on mobile; inline below questions on desktop. Last section: "Submit Assessment" primary Button replaces "Next Section".
- **Loading state for AI follow-ups:** A single Skeleton row (14px height, 60% width) appears below the triggering question while the AI generates the follow-up. Replaced by AiFollowUpQuestion on arrival.
- **Session persistence:** On each "Next Section" navigation, responses auto-saved to DB. No explicit "Save" action needed. User sees no save indicator — persistence is invisible.

### 4. Assessment Results — `/assessments/:id/results`

- **Layout:** Standard AppLayout with Breadcrumb. Single-column centered layout, max-width 800px.
- **Page header:** Assessment subject name + "Completed {date}" in muted 14px.
- **AiAnalysisCard** rendered first, full width. Shows AI narrative. "AI Analysis" Badge + AiProvenance (model, date) in card header. Card background uses `--muted` (hsl 240 4.8% 95.9% light) to stand out from white background. If AI summary is still generating, card shows Skeleton lines.
- **Score section header:** "Overall Score" at 20px/600. ScoreGauge centered below header, 200px diameter. Score numeral 28px/600 in center. Score tier label ("High" / "Medium" / "Low" / "Critical") below numeral in 14px/500.
- **Section scores:** "Section Breakdown" at 20px/600. List of SectionScoreBar rows, one per section, sorted by ascending score (worst first).
- **Per-question detail:** Collapsible per section. Section row header: section name + score. Expanded: list of question rows — question text, answer given, per-question score, weight. Per-question score omitted for non-numeric question types.
- **Action bar:** "Start New Assessment" (primary Button), "Export PDF" (outline Button, placeholder — shows "Coming soon" toast) — both in a row below score section.

### 5. Assessment List — `/assessments`

- **Layout:** Standard AppLayout. Page header: "Assessments" + "New Assessment" primary button.
- **Filter bar:** Status filter (All / In Progress / Completed), Subject type filter (All / Vendor / Compliance).
- **Table:** Columns — Subject, Template, Status badge, Overall Score, Completed date. Pagination consistent with existing list pages.
- **Empty state:** Heading "No assessments yet", body "Start by selecting a template from the library and assigning it to a vendor or compliance framework.", CTA "Browse Templates".

## Interaction Contracts

### Branching Logic (ASMT-02)
- Conditions evaluated client-side in real time during session.
- When a question is hidden by branching, it slides out with a 150ms height transition (CSS `max-height` 0 + `overflow hidden`).
- Hidden questions are not submitted — their answer slot is omitted from the response payload.
- In the template builder preview pane, branching conditions are not simulated (preview shows all questions). Branching simulation is session-only.

### AI Follow-up Questions (ASMT-05)
- Triggered after the user answers any question and moves focus away (onBlur for text; onChange for radio/checkbox/slider).
- Client sends current section responses to the backend. Backend returns 0 or 1 follow-up question per trigger.
- Follow-up appears with 200ms fade-in immediately below the triggering question.
- Follow-up questions are persisted to the session JSONB responses — they reappear identically on refresh.
- No more than 1 AI follow-up per question trigger per session (LLM decides whether to generate one; client renders whatever the backend returns, or nothing if empty).
- "AI Generated" Badge uses the existing `Badge` component with `variant="secondary"` and a `Sparkles` icon (h-3 w-3) prepended.

### Score Computation Display (ASMT-04)
- ScoreGauge animates from 0 to final score on mount (recharts animation prop, 800ms easeOut).
- SectionScoreBar bars animate left-to-right on mount (recharts animation, 600ms easeOut, staggered 100ms per bar).
- Score tier coloring: 80–100 = `--severity-low` (green), 60–79 = `--severity-medium` (yellow), 40–59 = `--severity-high` (orange), 0–39 = `--severity-critical` (red).

### Drag-to-Reorder Questions (ASMT-01)
- Use HTML5 `draggable` attribute + `onDragStart`/`onDrop` event handlers (no external DnD library). Simple vertical list reorder within a section.
- Drag handle: GripVertical lucide icon (h-4 w-4, muted color), 44px tall hit target.
- Dragging state: dragged item opacity 0.5, insertion indicator is a 2px solid `--primary` horizontal rule between rows.
