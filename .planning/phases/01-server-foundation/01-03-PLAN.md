  <action>
Read .env first to confirm DATABASE_URL and extract the password value. Then execute these commands IN ORDER — each step depends on the previous:

**Step 1: Install pgvector OS package (must be before any DB creation)**
```bash
sudo apt-get install -y postgresql-16-pgvector
```
Verify: `apt list --installed 2>/dev/null | grep pgvector` — should show postgresql-16-pgvector

**Step 2: Create database and dedicated user**
```bash
sudo -u postgres psql -c "CREATE DATABASE riskmind;"
sudo -u postgres psql -c "CREATE USER riskmind WITH PASSWORD 'riskmind';"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE riskmind TO riskmind;"
sudo -u postgres psql -d riskmind -c "GRANT ALL ON SCHEMA public TO riskmind;"
```
Note: The password 'riskmind' must match what is in .env's DATABASE_URL. If .env uses a different password, use that password here instead.

**Step 3: Install pgvector extension in the database (superuser required)**
```bash
sudo -u postgres psql -d riskmind -c "CREATE EXTENSION IF NOT EXISTS vector;"
```
Verify: `sudo -u postgres psql -d riskmind -c "SELECT extname FROM pg_extension WHERE extname='vector';"` — must return 'vector'

**Step 4: Set DATABASE_URL for drizzle-kit push**
Source the .env file to get DATABASE_URL into the environment, then run push:
```bash
cd /home/dante/RiskMind2 && export $(grep -v '^#' .env | xargs) && pnpm --filter @workspace/db run push
```
This runs drizzle-kit push with the DATABASE_URL from .env. Answer 'yes' to any confirmation prompts about applying schema changes.

**Step 5: Apply manual migration**

