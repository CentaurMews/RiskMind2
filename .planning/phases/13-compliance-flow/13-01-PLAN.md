**Types:**
```typescript
export interface RawRequirement {
  code: string;
  title: string;
  description?: string;
  parentCode?: string;
}

export interface DiffResult {
  new: RawRequirement[];
  modified: { incoming: RawRequirement; existing: { code: string; title: string; description?: string } }[];
  unchanged: RawRequirement[];
  warnings: string[];
}
```

**Functions:**

1. `parseCsv(fileBuffer: Buffer): RawRequirement[]`
   - Use `Papa.parse` with `{ header: true, skipEmptyLines: true, transformHeader: (h) => h.trim().toLowerCase() }`
   - Validate required fields: code and title must be non-empty strings
   - Throw descriptive error on parse failures or missing columns
   - Return cleaned array

2. `parseJson(fileBuffer: Buffer): RawRequirement[]`
   - Define Zod schema for JSON input: array of `{ code, title, description?, parentCode?, children?: recursive[] }`
   - Flatten nested hierarchy: if a node has `children`, recursively extract with `parentCode` set to parent's `code`
   - Validate with Zod, throw on invalid schema

3. `computeDiff(existing: { code: string; title: string; description?: string }[], incoming: RawRequirement[]): DiffResult`
   - Build Map from existing by code
   - For each incoming: if code not in existing -> "new"; if code in existing but title OR description differs -> "modified" (include both incoming and existing for diff display); else -> "unchanged"
   - Per D-05: this is additive-only — no "deleted" category. Existing requirements not in incoming are preserved (not returned in diff since no action needed)

4. `resolveParentCodes(requirements: RawRequirement[], codeToIdMap: Map<string, string>): { resolved: { code: string; title: string; description?: string; parentId: string | null }[]; warnings: string[] }`
   - For each requirement: look up parentCode in codeToIdMap; if found, set parentId; if parentCode is set but not found in map, set parentId to null and add warning
