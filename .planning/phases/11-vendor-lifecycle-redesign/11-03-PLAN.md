**Layout (per UI-SPEC section 6.1):**

**Step 1 — Identity + Tier (UI-SPEC 6.2):**
- Form with fields: name (required), description (Textarea 3 rows), category (Input), contactEmail (Input type=email), contactName (Input), tier (Select: Critical/High/Medium/Low with colored dots)
- On "Continue": if id is "new", POST /v1/vendors/onboard with form data, get back vendor with id, then navigate to /vendors/onboard/{id} and advance to step 2. If resuming, PATCH with step=1 data.
- Validate name is non-empty before submit. Show inline error "Vendor name is required." if empty.
- Validate email format if provided. Show "Enter a valid email address." if invalid.

**Step 2 — Questionnaire Assignment (UI-SPEC 6.3):**
- Fetch assessment templates via GET /v1/assessment-templates (use the existing Orval hook or fetch directly)
- Show template cards in 2-column grid. Each card: name (semibold), description (muted, 2-line clamp), question count badge
- Selected template gets border-primary ring-1 ring-primary
- "Skip for now" link below grid
- On "Continue": PATCH with step=2, data: { assessmentTemplateId } (or null if skipped)
- Empty state if no templates: "No Templates Available" / "Create an assessment template in Settings before assigning one here."

**Step 3 — Document Upload (UI-SPEC 6.4):**
- Styled file input with dashed border dropzone region (not a real dropzone library — just styled Input type=file)
- Upload icon + "Upload vendor documents" label centered
- Accepted types: PDF, DOCX, TXT shown below in text-xs text-muted-foreground
- After upload: show file list in Table (filename, size, remove button)
- Upload calls existing POST /v1/vendors/:vendorId/documents endpoint
- Below file list: LLM subprocessor suggestion panel. After document is uploaded, call POST /v1/vendors/:vendorId/extract-subprocessors. Show candidates with checkboxes. "Save Selected" saves checked items via POST /v1/vendors/:vendorId/subprocessors for each.
- "Continue" advances to step 4. No PATCH needed (documents are already uploaded).

**Step 4 — AI Enrichment Review (UI-SPEC 6.5):**
- Auto-trigger enrichment on step mount via useEffect: POST /v1/vendors/onboard/:id/enrich
- Show Loader2 spinner + "Analyzing vendor profile..." during polling
- Poll GET /v1/jobs/:jobId every 2 seconds until status is 'completed' or 'failed'
- On complete: show editable cards for each enrichment field (industry, risk indicators, breach history, description enrichment). Use Textarea/Input per field.
- On error: show Alert variant=destructive with "Enrichment failed. You can skip this step and complete onboarding." + Skip link
- "Complete Onboarding" button: PATCH with step=4 and confirmed enrichment data. Navigate to /vendors/:id after success.
- Idempotency: check if enrichment job already exists before triggering (the API handles this)

**Progress sidebar (UI-SPEC 6.1):**
- 4 steps: "Identity & Tier", "Questionnaire", "Documents", "AI Enrichment"
- Step circles: completed = bg-primary + CheckCircle2, active = bg-primary + number in white, upcoming = bg-muted + number in muted-foreground
- "Cancel Onboarding" link at bottom in text-destructive text-sm

**Cancel Onboarding (UI-SPEC section 7):**
- AlertDialog: title "Cancel Onboarding?", body "This will permanently delete the incomplete vendor record. This cannot be undone."
- Confirm "Delete Vendor" (destructive) calls DELETE /v1/vendors/onboard/:id, then navigates to /vendors
- Dismiss "Keep Editing" closes dialog

**SPA navigation guard (Pitfall 5):**
- Track isDirty state when form fields change within a step
- beforeunload event handler when isDirty
- For internal navigation: the wizard is self-contained, so guard is mainly for sidebar nav clicks. Use AlertDialog "Leave Onboarding?" per UI-SPEC.

**Resume flow:**
- On mount, if vendorId is not "new", fetch GET /v1/vendors/onboard/:id
- Set currentStep from response.wizardStep
- Pre-fill form fields from vendor data
- If vendor.status !== 'identification', redirect to /vendors/:id (wizard already completed)

**Imports:** Use shadcn components: Card, CardContent, Input, Textarea, Select, SelectContent, SelectItem, Badge, Progress, AlertDialog, Alert, Skeleton, Separator, Table, TableBody, TableCell, TableHead, TableRow, Button. Icons: ChevronRight, ChevronLeft, Plus, Loader2, Sparkles, Upload, CheckCircle2.
