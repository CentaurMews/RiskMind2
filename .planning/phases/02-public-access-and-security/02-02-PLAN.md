  <what-to-do>
Create a new dedicated tunnel in the Cloudflare dashboard. This CANNOT be automated — the dashboard UI generates the tunnel token.

Step-by-step:
1. Log into the Cloudflare dashboard at https://dash.cloudflare.com
2. Navigate to: Zero Trust → Networks → Tunnels
3. Click "Create a tunnel"
4. Choose "Cloudflared" as the connector type
5. Name the tunnel: `riskmind` (lowercase, no spaces)
6. Click "Save tunnel"
7. On the "Install and run a connector" screen, select:
   - Environment: Debian
   - Architecture: 64-bit
8. Copy the token from the `cloudflared service install <TOKEN>` command shown. The token is the long string after `--token` in that command. Do NOT run the command shown — only copy the token value.
9. Continue to the "Route tunnel" tab:
   - Public hostname: `app`
   - Domain: `riskmind.net`
   - Service type: `HTTP`
   - URL: `localhost:4000`
10. Click "Save hostname"
11. Verify in Cloudflare DNS that a CNAME record for `app.riskmind.net` was auto-created pointing to `<tunnel-uuid>.cfargotunnel.com`

Provide the token when resuming.

```bash

```
Confirm the output contains "pdpl.pulsebridge.me" or the existing pdpl token. If it does NOT (unexpected), stop and report before proceeding.

Write the new service file at `/etc/systemd/system/cloudflared-riskmind.service` with the token obtained in Task 1. Replace `RISKMIND_TOKEN_HERE` with the actual token value (no quotes around the token):

```ini
[Unit]
Description=cloudflared RiskMind tunnel
After=network-online.target
Wants=network-online.target

[Service]
TimeoutStartSec=30
Type=notify
ExecStart=/usr/local/bin/cloudflared --no-autoupdate --protocol http2 tunnel run --token RISKMIND_TOKEN_HERE
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

Then run:
```bash
systemctl daemon-reload
systemctl enable cloudflared-riskmind.service
systemctl start cloudflared-riskmind.service
```

Wait 5 seconds, then check status:
```bash
systemctl status cloudflared-riskmind.service
```

The status must show `active (running)`. If it shows `failed`, check `journalctl -u cloudflared-riskmind.service --no-pager -n 30` for the error and report.

CRITICAL: Do NOT modify or restart `cloudflared.service` at any point. The existing tunnel serving pdpl.pulsebridge.me must remain untouched.
