Run the following curl smoke tests in sequence. Report the result of each.

**Test 1 — Health endpoint via tunnel:**
```bash
curl -s -o /dev/null -w "Health via tunnel: %{http_code}\n" https://app.riskmind.net/api/v1/health
```
Expected: `200`

**Test 2 — CORS allowed origin:**
```bash
curl -s -I -H "Origin: https://app.riskmind.net" https://app.riskmind.net/api/v1/health
```
Expected: Response headers include `Access-Control-Allow-Origin: https://app.riskmind.net`

**Test 3 — CORS unauthorized origin rejected:**
```bash
curl -s -I -H "Origin: https://evil.com" https://app.riskmind.net/api/v1/health
```
Expected: Response headers do NOT include `Access-Control-Allow-Origin`

**Test 4 — CORS preflight from tunnel origin:**
```bash
curl -s -I -X OPTIONS \
  -H "Origin: https://app.riskmind.net" \
  -H "Access-Control-Request-Method: POST" \
  -H "Access-Control-Request-Headers: Authorization,Content-Type" \
  https://app.riskmind.net/api/v1/health
```
Expected: Response includes `Access-Control-Allow-Origin: https://app.riskmind.net` and `Access-Control-Allow-Credentials: true`

**Test 5 — SPA loads at root:**
```bash
curl -s -o /dev/null -w "SPA root: %{http_code}\n" https://app.riskmind.net/
```
Expected: `200`

If any test fails, investigate and fix before calling the human checkpoint. Common fixes:
- Health endpoint 502/503: `systemctl status cloudflared-riskmind.service` and `pm2 status`
- CORS headers missing: Verify `pm2 restart riskmind` was run after plan 02-01 changes
- Preflight failing: Confirm cors() middleware is BEFORE route handlers in app.ts (line 10 position)
