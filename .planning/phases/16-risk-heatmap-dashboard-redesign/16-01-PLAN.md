       - Accept optional query param `category` (string, one of riskCategoryEnum values) for domain filtering
       - Query active risks (status IN open, mitigated) for tenant, optionally filtered by category
       - Query latest snapshot (ORDER BY snapshotDate DESC LIMIT 1) for tenant
       - Query previous snapshot (ORDER BY snapshotDate DESC OFFSET 1 LIMIT 1) for delta computation
       - Query all appetite configs for tenant
       - Compute:
         - `postureScore`: compositeScore from latest snapshot, or compute live from active risks if no snapshot exists
         - `aboveAppetiteCount`: from latest snapshot, or compute live
         - `aboveAppetiteDelta`: difference from previous snapshot, or null
         - `cellDeltas`: Record<string, number> — diff between latest and previous snapshot cellCounts, or empty object
         - `domainSummaries`: Array of { category, displayName, score, count, highCriticalCount, sparklineData (last 6 months of scores from snapshots) }
         - `cells`: same as existing heatmap endpoint but with risk objects included
       - Category display name mapping: technology->Cyber, operational->Ops, compliance->Compliance, financial->Financial, strategic->Strategic, reputational->Reputational
       - Return JSON: `{ postureScore, aboveAppetiteCount, aboveAppetiteDelta, cellDeltas, aboveAppetiteCells (array of "l-i" keys), domainSummaries, cells, appetiteConfigs }`
       - If no snapshots exist, return `{ collecting: true, ...liveComputedValues }`

    2. **GET /v1/risks/snapshots** — Time-series for KRI trend panel:
       - Accept query param `range`: "3M" | "6M" | "12M" (default "6M")
       - Compute start date: subtract 3/6/12 months from today using date arithmetic
       - Query riskSnapshotsTable WHERE tenantId AND snapshotDate >= startDate ORDER BY snapshotDate ASC
       - Return JSON: `{ snapshots: Array<{ date: string, compositeScore: number, aboveAppetiteCount: number, totalRisks: number }> }`

       - Query all riskAppetiteConfigsTable rows for tenant
       - Return JSON: `{ configs: Array<{ category: string, threshold: number }> }`

    4. **PUT /v1/risks/appetite/:category** — Upsert appetite threshold (admin only):
       - Use `requireRole("admin")` middleware
       - Validate category is one of riskCategoryEnum values, threshold is integer 0-100
       - Upsert via `db.insert(riskAppetiteConfigsTable).values({ tenantId, category, threshold }).onConflictDoUpdate({ target: [riskAppetiteConfigsTable.tenantId, riskAppetiteConfigsTable.category], set: { threshold, updatedAt: new Date() } })`
       - After upsert, trigger a snapshot recapture for the tenant: `captureSnapshotForTenant(tenantId).catch(err => console.error("[Appetite] Snapshot recapture failed:", err))`
       - Return 200 with updated config
