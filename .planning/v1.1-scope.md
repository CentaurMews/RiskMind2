# v1.1 Milestone Scope — LLM Intelligence + Fixes + Polish

## Overview

Post-v1.0 milestone focused on three areas: intelligent LLM configuration/routing, fixing bugs discovered during audit, and polishing placeholder content.

## Phase 1: LLM Config Wizard & Model Router

### LLM Provider Onboarding Wizard
- Step 1: Nickname + provider type from dropdown (OpenAI, Anthropic, Google Gemini, Mistral, Groq, Together AI, Ollama/Private)
- Step 2: API key input (for private/Ollama: also base URL)
- Step 3: Connect and auto-fetch available models from provider API
  - OpenAI: `GET /v1/models`
  - Anthropic: hardcoded model list (no list endpoint)
  - Ollama: `GET /api/tags`
  - Others: provider-specific list endpoints
- Step 4: User picks models (can pick multiple for different use cases)
- Step 5: Test connection + benchmark (latency, tokens/sec, quality score via standard prompt)
- Step 6: Save config with model assignments

### Intelligent Model Router
- Routing table: 6 task types → model assignment
  - Risk Enrichment, Signal Triage, Treatment Suggestions, Embeddings, Agent Reasoning, General
- Smart defaults suggested based on benchmarks (cheapest fast model for triage, best reasoning for enrichment)
- Override per task type
- Visual routing table in Settings page
- Backend: extend `resolveConfig()` to accept task type parameter

## Phase 2: Bug Fixes from Audit

### Critical Fixes
1. **Document processing placeholder** — `doc-process` worker only reads filename, not file content. Either implement real document parsing (pdf-parse, mammoth) or show a clear "Document analysis coming soon" instead of hallucinated summaries.

2. **Agent local findings lost on LLM error** — When `reason()` throws, cascade/cluster/predictive findings are discarded. Fix: persist local findings before calling LLM `reason()`.

3. **Duplicate enrichment stacking** — Re-enriching a risk appends another `---AI Enrichment---` block. Fix: detect existing enrichment, replace instead of append.

4. **Vendor AI questions confusing 400** — LLM parse failure returns `"AI returned invalid format. Please try again."` which looks like a user error. Fix: return 502 or a clearer message.

### Quality Improvements
5. **Vendor scorecard real data** — `lastAssessmentDate` and `openFindingsCount` show placeholders. Either add these fields to the vendor API response or compute them from related tables.

6. **Embeddings health check** — Add a warning in Settings when no embeddings provider is configured (semantic search, agent clustering, signal correlation all silently degrade).

7. **Model name validation** — Prevent saving model names that don't match provider's format (the "Haiku" bug).
